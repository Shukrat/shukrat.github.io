Tower


The Project

Built using Monogame 3.1. Design and programming team of 4 people, including myself.

Tower is a build-your-own-map tower defence style game. Using the map editor provided with the game, the player is able to create a new map, place spawn portal, path and non-buildable location tiles. The player can save the map using a custom name, or load a previously saved one to edit further. Once saved, the player is free to share the map with others, or to play it on their own.



My Responsibilities - Programmer / User Interface Design

Programmer
The systems I programmed included: Game-state controller and system, map-editor in it's entirety, user text input system, save and load systems, user interface, and a small amount of gameplay algorithms.

Game-state Controller and System - In order to properly navigate the menus and run the game properly, my team and I discovered early on that we needed a significant game-state system. For this project, I had to learn and understand how a system like this would work, implement a game-state controller class, and test each screen-state after having programmed the system. Through this piece of the project, I was able to create a base code structure for a new game-state screen as needed, greatly reducing the amount of work necessary to add more to the project.

Map-editor - For this part, I programmed the tile selection, brush, save and load, and background selection systems. The save and load system utilizes a 2D array of integers, which assigned specified tiles to each integer. User Text Input System - For this part of the project, I contacted Liam Middlebrook (Git Hub) and worked with him to modify code he programmed for my own purposes. Using this system, the user is able to use custom map names to save and load, in addition to being able to see which maps are in the directory already.

User Interface Design
I designed and implemented the code for the menus, map-editor, and in-game user interfaces.
